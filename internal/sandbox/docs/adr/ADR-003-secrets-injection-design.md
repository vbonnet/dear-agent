# ADR-003: Secrets Injection Design

**Status**: Accepted
**Date**: 2026-03-20
**Authors**: Claude Sonnet 4.5
**Context**: Phase 1 - Core Implementation

## Context

AGM sessions need to inject API keys, tokens, and credentials into sandbox environments without:
1. Storing secrets in version control
2. Leaking secrets across sandbox boundaries
3. Persisting secrets in read-only repository layers
4. Requiring manual environment setup

## Decision

Implement **file-based secrets injection** that writes a `.env` file to the sandbox's `upperdir` with strict permissions and automatic cleanup.

### Design

**Location**: `{sandbox.UpperPath}/.env`

**Format**: Standard `.env` key-value pairs
```bash
# Generated by AGM Sandbox - DO NOT COMMIT
ANTHROPIC_API_KEY=sk-ant-...
GITHUB_TOKEN=${GITHUB_TOKEN}
DATABASE_URL=postgresql://...
```

**Permissions**: `0600` (owner read/write only, no group/other access)

**Environment Expansion**: Support `${VAR}` syntax for inheriting from parent

### Implementation

```go
func writeSecrets(upperDir string, secrets map[string]string) error {
    if len(secrets) == 0 {
        return nil // No .env file created
    }

    envPath := filepath.Join(upperDir, ".env")

    var content strings.Builder
    content.WriteString("# Generated by AGM Sandbox - DO NOT COMMIT\n")
    content.WriteString("# This file is isolated to this sandbox and will be removed on destroy\n\n")

    for key, value := range secrets {
        // Expand environment variables: ${VAR} → actual value
        expanded := os.ExpandEnv(value)
        content.WriteString(fmt.Sprintf("%s=%s\n", key, expanded))
    }

    // Write with restrictive permissions
    return os.WriteFile(envPath, []byte(content.String()), 0600)
}
```

## Consequences

### Positive

1. **Isolation**: Secrets only in upperdir, never in lowerdir
2. **Cleanup**: Automatic removal when sandbox destroyed
3. **Standard Format**: Compatible with most tools (.env files ubiquitous)
4. **Environment Expansion**: Allows inheriting from host without hardcoding
5. **Permissions**: Strict 0600 prevents unauthorized access
6. **Explicit**: .env file visible, easy to verify

### Negative

1. **File-Based**: Not as secure as encrypted keystores
2. **Plaintext**: Secrets readable by anyone with file access
3. **No Encryption**: Relies solely on filesystem permissions
4. **Agent Access Required**: Agent must source .env file

### Mitigations

- **Permissions**: 0600 ensures only session owner can read
- **Cleanup**: Destroy removes .env file (not persisted)
- **Upperdir Isolation**: Never written to lowerdir (no repo contamination)
- **Documentation**: Warn users about commit risks
- **Future**: Add encrypted secret store option (Phase 3+)

## Alternatives Considered

### 1. Environment Variables Only

```go
cmd := exec.Command("claude")
cmd.Env = append(os.Environ(), secretEnvVars...)
```

**Rejected**: Harder to debug, requires process tree inheritance

### 2. Encrypted Keychain

```go
keychain.StoreSecret("ANTHROPIC_API_KEY", value)
secret, _ := keychain.GetSecret("ANTHROPIC_API_KEY")
```

**Rejected**: OS-specific (macOS Keychain, Windows Credential Manager, Linux Secret Service), complex implementation

### 3. Config File with Encryption

```yaml
# .agm-secrets.yaml.enc (encrypted)
secrets:
  anthropic_api_key: <encrypted blob>
```

**Rejected**: Requires key management, adds complexity

### 4. HashiCorp Vault Integration

```go
client.Logical().Read("secret/data/agm/session")
```

**Rejected**: External dependency, overkill for local development

## Security Considerations

### Threat Model

**Protected Against**:
- Cross-sandbox leakage (upperdir isolation)
- Accidental commit (warning header, .gitignore pattern)
- Other users reading secrets (0600 permissions)
- Persistence after cleanup (Destroy removes .env)

**Not Protected Against**:
- Root user access (inherent Linux limitation)
- Process memory inspection (secrets in plaintext)
- Disk forensics after deletion (standard file system)
- Social engineering (user shares .env file)

### Best Practices

1. **Never Commit .env**: Warning header, .gitignore enforcement
2. **Rotate Secrets**: Regularly rotate API keys
3. **Scope Tokens**: Use minimum permissions (e.g., read-only GitHub token)
4. **Audit Logs**: Track secret access (future enhancement)

## Environment Variable Expansion

### Syntax

```go
secrets := map[string]string{
    "ANTHROPIC_API_KEY": "sk-ant-hardcoded",
    "GITHUB_TOKEN":      "${GITHUB_TOKEN}",        // Inherit from host
    "DATABASE_URL":      "${DB_HOST}/mydb",        // Partial expansion
    "COMBO":             "${VAR1}:${VAR2}",        // Multiple variables
}
```

### Behavior

- **Defined Variable**: `${GITHUB_TOKEN}` → actual token from host environment
- **Undefined Variable**: `${MISSING}` → empty string (Go os.ExpandEnv default)
- **Literal Dollar**: `$$` → `$` (escaped)

### Use Cases

1. **CI/CD Integration**: Inherit secrets from CI environment
2. **Development**: Use personal tokens from shell environment
3. **Testing**: Override with test credentials

## File Format Details

### Header

```bash
# Generated by AGM Sandbox - DO NOT COMMIT
# This file is isolated to this sandbox and will be removed on destroy
```

**Purpose**:
- Warn against committing
- Explain lifecycle (temporary)

### Key-Value Pairs

```bash
KEY=value
```

**Rules**:
- One per line
- No quotes around values (unless part of value)
- Supports special characters (URLs, JSON, etc.)

### Special Characters

```bash
DATABASE_URL=postgresql://user:pass@host/db  # OK
JSON_CONFIG={"key": "value"}                 # OK (no quoting needed)
PATH_WITH_SPACES=/path/with spaces           # OK (no escaping needed)
```

## Testing Strategy

### Unit Tests

```go
func TestWriteSecretsPermissions(t *testing.T) {
    upperDir := t.TempDir()
    secrets := map[string]string{"KEY": "value"}

    err := writeSecrets(upperDir, secrets)
    require.NoError(t, err)

    info, _ := os.Stat(filepath.Join(upperDir, ".env"))
    assert.Equal(t, os.FileMode(0600), info.Mode().Perm())
}
```

### Isolation Tests

```go
func TestSecretsIsolation(t *testing.T) {
    // Create two sandboxes with same lowerdir
    sb1, _ := provider.Create(ctx, req1)
    sb2, _ := provider.Create(ctx, req2)

    // Verify secrets not shared
    env1, _ := os.ReadFile(filepath.Join(sb1.UpperPath, ".env"))
    env2, _ := os.ReadFile(filepath.Join(sb2.UpperPath, ".env"))

    assert.NotEqual(t, env1, env2)
}
```

### Cleanup Tests

```go
func TestSecretsNotInLowerdirAfterDestroy(t *testing.T) {
    lowerDir := t.TempDir()
    sb, _ := provider.Create(ctx, SandboxRequest{
        LowerDirs: []string{lowerDir},
        Secrets:   map[string]string{"KEY": "value"},
    })

    provider.Destroy(ctx, sb.ID)

    // Verify .env not in lowerdir
    _, err := os.Stat(filepath.Join(lowerDir, ".env"))
    assert.True(t, os.IsNotExist(err))
}
```

## Integration with AGM

### Manifest

```go
type Manifest struct {
    Sandbox *SandboxConfig `json:"sandbox,omitempty"`
}

type SandboxConfig struct {
    Enabled  bool              `json:"enabled"`
    Secrets  map[string]string `json:"secrets,omitempty"`
}
```

**Note**: Secrets stored in manifest are expanded and written to .env

### Config

```yaml
sandbox:
  enabled: true
  secrets:
    ANTHROPIC_API_KEY: ${ANTHROPIC_API_KEY}
    GITHUB_TOKEN: ${GITHUB_TOKEN}
```

**Workflow**:
1. User sets secrets in config or via flags
2. AGM passes secrets to `sandbox.Create()`
3. Provider writes .env to upperdir
4. Agent sources .env in session

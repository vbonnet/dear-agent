## 2. Color Contrast Analysis (WCAG AA Compliance)

### 2.1 Test Methodology

Contrast ratios calculated using WCAG 2.1 formula:
```
Contrast Ratio = (L1 + 0.05) / (L2 + 0.05)
where L = relative luminance
```

**WCAG Requirements:**
- AA Normal Text: 4.5:1 minimum
- AA Large Text (≥18pt): 3.0:1 minimum
- Non-text elements: 3.0:1 minimum

### 2.2 Color Palette Analysis

**Primary C4 Color Scheme (D2/Mermaid):**

| Color Name | Hex Code | Use Case |
|------------|----------|----------|
| Person | `#08427b` | User/actor shapes |
| System | `#1168bd` | Software systems |
| Container (light) | `#438dd5` | Containers, databases |
| Container (dark) | `#1168bd` | Components |
| External System | `#999999` | External dependencies |
| Queue | `#ff9f43` | Message queues |
| Cache | `#ff6b6b` | Cache systems |

### 2.3 Contrast Ratios vs White Background

| Color | vs White | AA Normal | AA Large | AAA Normal | Status |
|-------|----------|-----------|----------|------------|--------|
| Person (#08427b) | 10.12:1 | ✓ | ✓ | ✓ | EXCELLENT |
| System (#1168bd) | 5.62:1 | ✓ | ✓ | ✗ | GOOD |
| Container Light (#438dd5) | 3.49:1 | ✗ | ✓ | ✗ | ACCEPTABLE (Large) |
| External (#999999) | 2.85:1 | ✗ | ✗ | ✗ | **FAIL** |
| Queue (#ff9f43) | 2.04:1 | ✗ | ✗ | ✗ | **FAIL** |
| Cache (#ff6b6b) | 2.78:1 | ✗ | ✗ | ✗ | **FAIL** |

### 2.4 Text on Colored Backgrounds

**Critical combinations tested:**

| Combination | Ratio | Status | Notes |
|-------------|-------|--------|-------|
| White on Person (#08427b) | 10.12:1 | ✓ PASS AA | Excellent contrast |
| White on System (#1168bd) | 5.62:1 | ✓ PASS AA | Good contrast |
| Black on Light Blue (#438dd5) | 6.01:1 | ✓ PASS AA | Good contrast |
| Black on Gray (#999999) | 7.37:1 | ✓ PASS AA | Excellent contrast |
| Black on Orange (#ff9f43) | 10.29:1 | ✓ PASS AA | Excellent contrast |
| Black on Red (#ff6b6b) | 7.57:1 | ✓ PASS AA | Excellent contrast |

### 2.5 Assessment Summary

**Status:** ⚠ **PARTIAL PASS**

**Strengths:**
- ✓ Text on colored backgrounds: All combinations pass AA
- ✓ Primary colors (Person, System): Excellent contrast
- ✓ Dark blue tones: Meet AAA standards

**Issues Identified:**
1. **External System Gray (#999999)**: 2.85:1 vs white - Below AA threshold
   - Used for: External dependencies, third-party systems
   - Impact: Medium contrast may be difficult for low-vision users

2. **Queue Orange (#ff9f43)**: 2.04:1 vs white - Below AA threshold
   - Used for: RabbitMQ, message queues
   - Impact: Light orange is hard to distinguish from background

3. **Cache Red (#ff6b6b)**: 2.78:1 vs white - Below AA threshold
   - Used for: Redis cache, caching systems
   - Impact: Light red lacks sufficient contrast

**Root Cause:** Accent colors prioritize aesthetics over contrast

### 2.6 Recommendations

#### Immediate Fixes (High Priority)

**Option 1: Darken Problematic Colors**
```d2
# Current
external: { style.fill: "#999999" }  # 2.85:1
queue:    { style.fill: "#ff9f43" }  # 2.04:1
cache:    { style.fill: "#ff6b6b" }  # 2.78:1

# Recommended (AA compliant)
external: { style.fill: "#666666" }  # 5.74:1 ✓
queue:    { style.fill: "#cc7a00" }  # 4.52:1 ✓
cache:    { style.fill: "#d63031" }  # 4.51:1 ✓
```

**Option 2: Add Stroke Borders**
```d2
# Add dark borders for better definition
external: {
  style.fill: "#999999"
  style.stroke: "#333333"
  style.stroke-width: 2
}
```

**Option 3: Use Patterns/Textures**
```d2
# Combine color with visual patterns
queue: {
  style.fill: "#ff9f43"
  style.stroke-dash: 5  # Dashed outline for distinction
}
```

#### Long-term Improvements

1. **Create Accessibility-First Color Palette**
   - Design new palette with 4.5:1 minimum contrast
   - Maintain C4 semantic meaning (blue=system, gray=external)
   - Test with colorblind simulators

2. **Add High-Contrast Mode**
   - Provide alternate color scheme (black, white, high-contrast primaries)
   - Example: `d2 --theme high-contrast diagram.d2 output.svg`

3. **Document Color Standards**
   - Add WCAG contrast requirements to style guide
   - Provide color picker tool with contrast checker
   - Include accessibility testing in CI/CD

## 6. Critical Issues Identified

### Issue 1: Low Contrast Accent Colors (HIGH PRIORITY)

**Severity:** HIGH
**Impact:** Users with low vision, aging eyes, or viewing in bright light
**Affected Elements:** External systems, queues, caches

**Current State:**
- External (#999999): 2.85:1 vs white
- Queue (#ff9f43): 2.04:1 vs white
- Cache (#ff6b6b): 2.78:1 vs white

**Required Fix:** Darken colors to achieve 4.5:1 minimum

**Recommended Palette:**
```d2
external: { style.fill: "#666666" }  # 5.74:1
queue:    { style.fill: "#cc7a00" }  # 4.52:1
cache:    { style.fill: "#d63031" }  # 4.51:1
```

### Issue 2: Missing SVG Accessibility Attributes (MEDIUM PRIORITY)

**Severity:** MEDIUM
**Impact:** Screen reader users
**Affected Elements:** All generated SVG outputs

**Current State:** Unknown (pending verification)

**Required Fix:**
1. Ensure `<title>` and `<desc>` tags in SVG
2. Add ARIA labels to grouped elements
3. Include `lang` attribute on root `<svg>`

**Verification Command:**
```bash
xmllint --noout /tmp/test-diagrams/c4-context.svg
grep -i "title\|desc\|aria" /tmp/test-diagrams/c4-context.svg
```

### Issue 3: Color as Sole Indicator (LOW PRIORITY)

**Severity:** LOW
**Impact:** Colorblind users (8% of males, 0.5% of females)

**Current State:**
- External systems: Gray color only
- Queues: Orange color only
- Caches: Red color only

**Recommendation:**
- Add shape variations (hexagon for queues, diamond for caches)
- Include icon/pattern fills
- Ensure labels always present

**Example Enhancement:**
```d2
queue: {
  shape: hexagon        # Unique shape
  label: "RabbitMQ"
  style.fill: "#cc7a00"
  style.stroke-dash: 5  # Pattern
}
```

# SPEC: generate-patterns

Code generator that reads the unified YAML pattern database and emits the
`internal/validator/patterns.go` Go source file consumed by `pretool-bash-blocker`.

## Purpose

`patterns/bash-anti-patterns.yaml` is the single source of truth for all bash
anti-patterns. `generate-patterns` bridges the YAML database to the Go validator
by generating a `.go` file containing three parallel arrays (`forbiddenPatterns`,
`patternNames`, `remediations`) that the validator loads at package init with
zero per-request overhead.

## Input: bash-anti-patterns.yaml

Each entry in the `patterns` list supports these fields relevant to generation:

| Field              | Type     | Required | Description                                      |
|--------------------|----------|----------|--------------------------------------------------|
| `id`               | string   | yes      | Unique identifier                                |
| `order`            | int      | yes      | Sort key; first match wins in the validator      |
| `re2_regex`        | string   | yes*     | RE2-compatible regex; absent = excluded          |
| `pattern_name`     | string   | yes      | Human-readable name, written as a code comment   |
| `remediation`      | string   | yes      | Actionable fix text emitted in block messages    |
| `examples`         | []string | no       | Strings that **must** match the regex (lint)     |
| `should_not_match` | []string | no       | Strings that must **not** match the regex (lint) |
| `relaxed`          | bool     | no       | `true` = excluded from generation                |

Patterns without `re2_regex` or with `relaxed: true` are filtered out silently
before generation. The 54-pattern database (as of commit 543cb4ed) yields 29
active patterns after filtering.

## Output: patterns.go

Generated into `hooks/internal/validator/patterns.go`. Skeleton:

```go
// Code generated by generate-patterns; DO NOT EDIT.
// Source: patterns/bash-anti-patterns.yaml

package validator

import "regexp"

var forbiddenPatterns = []*regexp.Regexp{ ... }
var patternNames      = []string{ ... }
var remediations      = []string{ ... }
```

Patterns are sorted by `order` ascending. Index position is shared across all
three arrays. Regex literals use raw backtick quoting by default; if the regex
itself contains a backtick, double-quote mode with `\x60` escaping is used.

## Flags

```
-yaml   <path>   Path to bash-anti-patterns.yaml  (default: ../../patterns/bash-anti-patterns.yaml)
-output <path>   Destination file                  (default: stdout)
--lint           Validate only; do not write output (exits 0 on success, 1 on error)
```

## Validation (always runs before generation)

1. Every active pattern must have non-empty `pattern_name` and `remediation`.
2. `re2_regex` must compile under Go's `regexp` package (RE2 semantics).
3. `order` values must be unique across all active patterns.

Validation errors are printed to stderr and cause exit code 1.

## Lint Mode (`--lint`)

Runs validation **plus** example testing:

- Every string in `examples` must match the regex.
- Every string in `should_not_match` must not match.

Exits 0 with `ok: N patterns validated, all examples pass` on stderr on success.
Exits 1 with per-pattern diagnostics on failure. Never writes output.

## How to Run

Regenerate after any change to `bash-anti-patterns.yaml`:

```sh
# From hooks/
make generate
# Equivalent direct invocation:
go run ./cmd/generate-patterns \
  -yaml ../patterns/bash-anti-patterns.yaml \
  -output internal/validator/patterns.go
```

Validate the pattern database without writing (CI):

```sh
make lint-patterns
# Equivalent:
go run ./cmd/generate-patterns -yaml ../patterns/bash-anti-patterns.yaml --lint
```

Assert the committed file matches a fresh generate run (CI gate):

```sh
make check-generate    # runs generate then git diff --exit-code
```

## go:generate Integration

`hooks/internal/validator/doc.go` contains:

```go
//go:generate go run ../../cmd/generate-patterns -yaml ../../../patterns/bash-anti-patterns.yaml -output patterns.go
package validator
```

Running `go generate ./internal/validator/` (invoked by `make generate`)
regenerates `patterns.go` in place. The file carries a `DO NOT EDIT` header;
manual edits are overwritten on the next generate run.

## Tests

`main_test.go` provides 22 test cases covering `validate()`, `lintExamples()`,
`regexLiteral()`, `goStringLiteral()`, and `generate()`, plus a round-trip
integration test that reads the real YAML, generates output, and byte-compares
it against the committed `patterns.go`.
